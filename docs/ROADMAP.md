# 로드맵

## 개발 순서

웹앱(그릇)을 먼저 만들고, 익스텐션(담는 도구)을 나중에 붙인다.

---

## 1단계 — 웹앱 대시보드 (현재)

**목표**: localStorage 기반으로 직접 입력 + 관리가 되는 MVP

- [ ] 타입 정의 (`types/application.ts`)
- [ ] Zustand 스토어 + localStorage 연동
- [ ] 칸반 보드 UI (5개 컬럼)
- [ ] 지원 카드 컴포넌트
- [ ] 카드 추가/수정/삭제
- [ ] 드래그로 상태 변경 (dnd-kit)
- [ ] 일기 기능 (카드별 일기 작성)
- [ ] 통계 요약 (총 지원/합격률)
- [ ] 텍스트 복사 기능
- [ ] 이미지 카드 공유
- [ ] 반응형 (모바일)
- [ ] Vercel 배포

## 2단계 — 크롬 익스텐션

**목표**: 사람인부터 지원 내역 자동 수집

- [ ] `jobdiary-extension` 레포에 CRXJS 세팅
- [ ] manifest.json 작성
- [ ] 사람인 content script (셀렉터 찾기 → 수집)
- [ ] service worker (저장 로직)
- [ ] popup UI
- [ ] 원티드, 잡코리아 확장
- [ ] 로컬 테스트

## 3단계 — 연동

**목표**: 익스텐션이 수집한 데이터를 웹앱에 자동 반영

- [ ] 익스텐션 → 웹앱 데이터 전달
- [ ] 중복 처리
- [ ] 크롬 웹스토어 배포

## 4단계 — 로그인 + 동기화

**목표**: 여러 기기 동기화

- [ ] Supabase 연동
- [ ] 로그인 (소셜)
- [ ] 데이터 마이그레이션 (localStorage → DB)

---

## KPI

| 시점 | 목표 |
|------|------|
| 1개월 | 익스텐션 설치 100명 |
| 3개월 | MAU 500명, 웹스토어 별점 4.5+ |
| 6개월 | MAU 2,000명, 프리미엄 전환 시작 |
