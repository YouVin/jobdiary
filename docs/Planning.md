# 기획서

## 프로젝트: 취준일기 (JobDiary)

---

## 1. 문제

취준생은 사람인, 원티드, 잡코리아 등 여러 플랫폼에 동시에 지원한다. 하지만 각 플랫폼은 자기 사이트의 지원 내역만 보여줘서, "내가 어디에 지원했고 지금 어떤 상태인지"를 한눈에 볼 수 없다. 노션이나 엑셀로 직접 정리하려 하면 건수가 쌓일수록 입력이 귀찮아져 결국 포기한다.

## 2. 해결책

크롬 익스텐션이 채용 사이트에서 지원 내역을 자동으로 수집해 웹앱 대시보드에 기록한다. 유저는 직접 입력 없이 칸반 보드로 전체 현황을 관리하고, 각 지원 건마다 일기를 남겨 취업 여정을 기록한다.

**"지원만 하면 자동으로 정리되고, 취업 여정이 일기가 된다"**

## 3. 시장 검증

### 해외엔 이미 검증된 시장
- **Simplify Copilot**: 100만+ 유저, 별점 4.9, 무료 + 프리미엄
- **Teal, Huntr, JobWizard**: 동일 카테고리 경쟁자 다수
- 공통 구조: 칸반 트래커 + 자동 수집 + 로컬 저장

### 한국 시장은 공백
위 도구들은 전부 미국 ATS(Workday, Greenhouse) 기반. 사람인·잡코리아·원티드 대응 도구는 없다.

## 4. 차별화 — 일기 기능

해외 트래커는 데이터 관리만 한다. 취준일기는 **감정 기록 + 회고**까지 담아 한국 취준생 정서에 맞춘다. 취준은 멘탈 싸움이기 때문이다.

- 지원 카드마다 일기 첨부 ("오늘 면접 망한 듯 ㅠㅠ", "서류 붙음!!")
- 합격 시 그동안의 일기가 하나의 취업 성공 스토리로
- 이미지 공유 시 "23곳 지원, 5개월의 기록" 감성 스토리텔링

## 5. 타겟

- 1차: 동시 다발적으로 지원하는 취준생 (신입/경력)
- 2차: 이직 준비 중인 재직자

## 6. 수익 모델

```
1단계 (런칭~3개월): 완전 무료 → 유저 모으기
2단계 (트래픽 확보): 광고 + 프리미엄
```

프리미엄 후보: 무제한 저장, AI 자소서 첨삭, 면접 알림, 상세 통계, 광고 제거 (월 990~1,990원)

## 7. 제외 범위 (MVP 아님)

- 채용 공고 검색 (각 플랫폼이 이미 잘함)
- 자동 지원 (약관 위험)
- 이력서 자동 작성
- 모바일 앱

## 8. 리스크 & 대응

| 리스크 | 대응 |
|--------|------|
| 사이트 HTML 변경으로 수집 깨짐 | 수동 저장 버튼 제공, 셀렉터 모듈화 |
| 로그인 정보 보안 우려 | 로그인 정보 서버 전송 안 함, 로컬 처리만 |
| 웹스토어 심사 거절 | 권한 최소화, 개인정보 정책 명확화 |
| 초기 트래픽 부족 | 취준 커뮤니티 + 유튜브/블로그 바이럴 |
